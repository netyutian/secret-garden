# 花园地块一键操作设计

## 背景

当前游戏中，点击地块会弹出一个操作菜单（或依赖底部工具栏选择工具），用户需要多做一步选择才能执行开荒、浇水、凋谢、铲除等操作。本次设计目标是把这四个常用操作变成"点击地块直接生效"，同时优化 hover 提示。

## 目标

1. 点击地块直接执行状态对应的默认操作，无需先选择工具或打开菜单。
2. 鼠标悬停在地块上时，提示即将执行的操作名称。
3. 播种仍需要选择花种，但点击耕地后直接弹出花种选择器。

## 方案

采用"全自动模式"：完全根据地块状态决定点击行为，移除锄头、水壶、铲子、手等工具按钮。

## 地块点击行为

| 地块状态 | 点击结果 |
|---------|---------|
| `wild`（荒地） | 执行开荒 → `tilled` |
| `tilled`（耕地） | 弹出花种选择器，选花后播种 → `seed` |
| `seed`（种子） | 执行浇水 → `sprout` |
| `sprout`（发芽） | 执行浇水 → `growing` |
| `growing`（成长） | 执行浇水 → `blooming` |
| `blooming`（盛开） | 执行凋谢 → `withered` |
| `withered`（凋谢） | 执行铲除 → `wild` |

### 资源校验

- 浇水时若水滴不足，保持现有行为：播放错误音效并提示"水滴不足"，地块状态不变。
- 铲除、凋谢等操作不消耗资源。

## 提示优化

每个地块的 `title` 属性动态设置为当前状态对应的操作名：

| 地块状态 | 提示文本 |
|---------|---------|
| `wild` | 开荒 |
| `tilled` | 播种 |
| `seed`/`sprout`/`growing` | 浇水 |
| `blooming` | 凋谢 |
| `withered` | 铲除 |
| `locked` | 解锁 |

提示使用浏览器原生 `title` 实现，简洁且无需额外 DOM。

## 工具栏调整

- 移除底部工具栏中的锄头、水壶、铲子、手四个按钮。
- 保留种子选择逻辑，但改为由耕地地块点击触发。
- 原有 `activeTool` 状态不再用于决定地块操作，但保留 `selectedFlowerId` 用于播种。

## 代码改动点

- `src/ui/garden-view.ts`
  - 重写 `handlePlotClick`：按地块状态直接执行对应操作。
  - 删除 `showMenu` 中针对四操作的菜单按钮，仅保留耕地的花种选择器入口。
  - 在 `renderPlots` 中为每个地块设置 `title` 属性。
- `src/ui/toolbar.ts`
  - 移除锄头、水壶、铲子、手按钮，保留种子选择器（或整体移除，视耕地点击触发方式而定）。
- `src/game/state.ts`
  - 复用现有 `advancePlot`、`plantFlower`、`witherPlot`，无需新增状态逻辑。
  - 可删除 `setTool` 与 `Tool` 类型中不再使用的工具，但为最小化改动，先保留类型定义。
- `src/style.css`
  - 保持现状。若后续移除工具栏，可清理相关样式；浇水光标 `.tool-water` 可暂时保留或移除。

## 数据流

1. 用户点击地块。
2. `handlePlotClick` 读取当前地块状态。
3. 根据状态调用 `advancePlot(state, plotId, 'hoe' | 'water' | 'shovel')`、`plantFlower` 或 `witherPlot`。
4. 通过 `onChange` 更新全局状态。
5. `updateGarden` 重新渲染地块。

## 测试

- 单元测试：验证各状态地块点击后的状态流转、水滴消耗、金币奖励。
- 手动验证：
  - 荒地点击后变为耕地。
  - 耕地点击后弹出花种选择器，选花后变为种子。
  - 种子/发芽/成长点击后进阶。
  - 盛开点击后凋谢并获得金币。
  - 凋谢点击后铲除并变为荒地。
  - 水滴不足时浇水提示错误。
  - 悬停提示显示正确操作名。

## 未包含范围

- 锁定地块的解锁交互保持不变：点击锁定地块仍弹出解锁菜单。
- 图鉴、背景音乐、水源等其它 UI 不变。
